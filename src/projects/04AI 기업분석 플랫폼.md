**진행 기간**: 2025.06.27. ~ 2025.07.04. 8일 (미니 팀 프로젝트)

## 개요

DB SQL 쿼리, React 프론트엔드 등의 풀스택 교육 과정을 이수한 후, CRUD 기능을 활용한 비즈니스 서비스 구축을 목적으로 진행한 프로젝트입니다. DART Open API와 Naver News API로 수집한 기업 재무 데이터 및 최신 뉴스를 바탕으로, Groq AI API를 통해 기업 핵심 정보를 요약해주는 사용자 맞춤형 웹 애플리케이션입니다.

## 역할

- **프로젝트 매니징 및 협업 환경 구축**: GitHub 조직(Organization) 개설 및 기능 단위 브랜치 분기/병합(PR) 컨벤션 도입.
- **Frontend 아키텍처 및 프로토타입 설계**: React 기반 TypeScript 웹앱 초기 환경 설정 및 더미 데이터 연동 프로토타입 구축.
- **인증 및 인가 처리**: Google OAuth 기반 사용자 로그인 구현 및 JWT 세션 유지 로직 개발.
- **백엔드 DB 모델링 및 사용자 CRUD 개발**: FastAPI 연동 사용자 정보(관심 카테고리, 선호 기업 등) DB 모델링 및 API 구현.
- **AI 기업 요약 및 검색 시스템 구현**: 팀 데이터 기반 Groq AI 기업 요약 기능 통합 및 클라이언트 사이드 기업 검색 로직 구현.

## Skills

- **Frontend**: React, TypeScript
- **Backend & DB**: Python, FastAPI, SQLAlchemy, MySQL
- **AI & API**: Groq AI API(gemma 모델), DART Open API, Naver News API

### 2. **프론트엔드 프로토타입 및 검색 로직 구현**

![프로토타입 웹 UI(더미데이터)](https://i.postimg.cc/xdjDJb7T/image-1.png)

백엔드 API 연동 전 프론트엔드 개발 속도를 높이기 위해 더미 데이터를 활용한 React TypeScript 기반 초기 웹앱 뼈대를 구축했으며, 별도의 백엔드 검색 엔진 없이 클라이언트 사이드에서 동작하는 기업 검색 기능을 자체 구현했습니다.

Google OAuth를 활용한 사용자 로그인 기능을 구현하고 토큰 기반의 인가 로직을 구성했습니다. 또한 백엔드 서버에서 사용자 관심 기업 카테고리 및 선호 기업 통계(하트 버튼 연동)를 관리하는 DB 모델링과 API를 설계했습니다.

### 4. **생성형 AI 기반 요약 자동화**

팀원들이 수집한 DART 재무제표 정보와 카테고리별 최신 뉴스 데이터를 프롬프트로 엮어, Groq AI가 해당 기업의 핵심 정보를 즉시 요약해주는 파이프라인을 완성했습니다.

### 5. OAuth 콜백 리디렉션 및 CORS 이슈 해결

#### AS-IS
사용자 Google 로그인 성공 후 대시보드 화면으로 리디렉션되지 않고, 백엔드 콜백 라우터가 반환한 JSON 응답 화면(API 페이지)에 브라우저가 그대로 머무르는 버그가 발생했습니다.

#### TO-BE
FastAPI 응답을 `starlette.responses.RedirectResponse`로 변경하여 프론트엔드 URL과 세션 토큰을 함께 넘겨주도록 수정했습니다. 동시에 `CORSMiddleware`를 추가하여 교차 출처 리소스 공유(CORS) 문제를 해결하고 정상적인 페이지 이동을 구현했습니다.

### 6. 프론트엔드 상태 관리 및 렌더링 최적화

#### AS-IS
기업 상세 페이지 로드 시 기업 요약, 재무, 뉴스 컴포넌트들이 각기 다른 타이밍에 개별 API를 호출하여, 화면 렌더링이 일괄적으로 처리되지 않고 시각적 깜빡임(지연)이 발생했습니다.

#### TO-BE
데이터 페칭 로직을 `useCompanyDetail` 커스텀 훅(Hook)으로 통합했습니다. 훅 내부에서 모든 API 데이터를 완전히 로딩한 뒤 단일 상태(State)에 담아 전달함으로써, 페이지 컴포넌트가 한 번에 렌더링되도록 UX를 최적화했습니다.

### 7. DART API 이기종 데이터 검증 및 예외 처리

#### AS-IS
DART API에서 수집해 DB에 적재한 기업 개황 정보 중, 누락된 필드가 있거나 재무제표 데이터 배열이 불규칙한 경우 프론트엔드에서 데이터를 호출할 때 잦은 에러가 발생했습니다.

#### TO-BE
DB에서 값을 꺼내 클라이언트로 전달하기 전, 동일한 데이터 구조로 파싱하는 미들웨어 형태의 데이터 검증 라우터를 백엔드에 구축하고 예외 처리를 적용하여 서버 안정성을 확보했습니다.

### 8. 불필요한 더미 데이터 및 노이즈 필터링

#### AS-IS
DART 개황 정보 원본에 "O호 스팩", "OO기업인수목적" 등 분석 대상이 아닌 페이퍼 컴퍼니 성격의 데이터가 다수 섞여 있고, 기업명에 "(주)", "주식회사" 등의 접두사가 포함되어 사용자 검색 경험이 저하되었습니다.

#### TO-BE
사용자 UI 및 검색 로직에 정규 표현식(Regex)을 적용하여 스팩(SPAC) 기업들이 리스트에 노출되지 않도록 필터링하고, 불필요한 회사명 키워드를 일괄 치환하여 검색 정확도와 데이터 가독성을 높였습니다.

## 성과 및 회고

### 성과

1. 주말을 제외한 약 5일간의 짧은 개발 기간 내에 React 프론트엔드와 FastAPI 백엔드를 결합한 기업 분석 플랫폼 배포를 완료했습니다.
2. 기능별 브랜치 분기 및 PR 컨벤션, GitHub Issues를 활용한 기능단위 작업 관리를 프로젝트에 도입하여 운영했습니다.

### 회고

1. **협업 도구로서의 GitHub 활용**: 과거 프로젝트에서 단일 브랜치 작업 시 겪었던 극심한 병합 충돌 문제를, 이번 프로젝트에서 체계적인 브랜치 전략과 이슈 관리를 도입함으로써 완벽하게 통제할 수 있었습니다. 주말 동안 미리 레포지토리 전략과 초기 웹 프로토타입 뼈대를 세워둔 것이 5일이라는 짧은 기간 내에 프로젝트를 완수할 수 있었던 핵심 요인이라고 판단합니다.
2. **데이터 확장에 대한 아쉬움**: 현재 DART 기업 개황 데이터와 뉴스에만 의존하고 있으나, 향후 외부 API를 연계하여 직원 수, 평균 연봉, 현직자 기업 후기 등의 데이터를 추가 확보한다면 훨씬 실용적인 분석 정보를 제공할 수 있을 것입니다.
3. **추천 시스템 고도화 방향**: 사용자의 선호 기업 및 관심 카테고리 DB가 이미 구축되어 있으므로, 현재 채용이 진행 중인 공고 API를 추가로 연동한다면 '개인 맞춤형 채용 공고 추천 시스템'으로 서비스를 크게 고도화할 수 있을 것으로 전망합니다.
