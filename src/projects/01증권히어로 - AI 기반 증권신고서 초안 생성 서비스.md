## 개요

작성하기 까다로운 증권신고서의 진입 장벽을 낮추고, 잦은 정정 요청으로 인해 발생하는 막대한 비용과 리소스를 절감하기 위해 기획된 LangGraph 기반 멀티 에이전트 시스템입니다. 주로 증권사 주니어 직원이 수행하는 수백 페이지 분량의 증권신고서 작성 시 발생하는 잦은 실수를 방지하고, 대량의 DART 공시 정보를 기반으로 증권신고서 초안의 약 70%를 자동 생성하여 업무 효율성을 극대화하는 것을 목적으로 합니다.  또한, 엔터프라이즈급 경험을 위해 대용량 트래픽을 고려한 MSA 아키텍처를 적용했습니다.

![시스템 아키텍처 구조도](https://res.cloudinary.com/dcbaxgous/image/upload/q_auto/f_auto/hwamyeon-kaebcheo-2026-03-25-220114_iqbjte.png)

## 역할 (AI 백엔드)

- DART 공시 서류 파싱 및 OpenSearch 인덱싱 파이프라인 설계
- LangGraph 기반 AI 멀티 에이전트 아키텍처 및 비동기 처리 로직 설계
- 토큰 감축 및 응답 정확도 향상을 위한 프롬프트 엔지니어링 및 시스템 프롬프트 동적 할당
- 노이즈(불용어, 수치, 중복 텍스트 등) 제거 및 가중치 기반의 지침 검색 알고리즘 설계
- 생성된 초안의 검토 및 합/불(PASS/FAIL) 판단을 위한 피드백 루프(검증 노드) 설계
- 신규 기술 스택(Spring Boot, OpenSearch, Kafka 등) 사내 스터디 주도 및 노션 문서화

## Skills

- **Frontend**: React, TypeScript, Tailwind CSS
- **Backend & AI**: Java, Spring Boot, Spring AI, LangGraph, Python, FastAPI, RAG, MCP
- **Data & Search**: OpenSearch (ElasticSearch), Kafka, MariaDB
- **Infrastructure**: AWS, Kubernetes (k8s), Docker, Terraform, Github Action

## 주요 수행 내용 및 문제 해결 과정

### 1. DART 공시서류 데이터 파이프라인 구축 및 OpenSearch 인덱싱

#### AS-IS

AI가 정확한 지침과 레퍼런스를 참조하여 초안을 생성하려면, 방대한 DART 공시 서류들을 AI가 처리하기 쉬운 형태로 가공하고 저장할 효율적인 데이터 파이프라인이 부재했습니다.

#### TO-BE  [🔗 [Github Repo](https://github.com/duozone-1-finalproject/Utility) ]

OpenSearch 인덱스 매핑을 설계하고, Python으로 DART XML 파일을 파싱하는 FastAPI 서버를 구축했습니다. Spring Boot 메인 서버의 요청에 따라 대상 기업의 보고서를 자동 검색, 다운로드, 파싱하여 즉시 적재하는 자동화 파이프라인을 완성했습니다. 공시 문서의 특성을 고려해 벡터 DB 대신 목차와 소제목 단위 평문 텍스트 매핑 방식을 채택했습니다.

### 3. AI 할루시네이션 방지 및 토큰 최적화를 위한 검색/프롬프트 전략

#### AS-IS

방대한 지침 문서와 레퍼런스를 RAG에 그대로 활용한 결과, 토큰 제한을 초과하고 할루시네이션이 발생하여 응답 품질이 저하되었습니다.

#### TO-BE

OpenSearch 검색 시 산업군, 지리적 특성에 가중치를 부여하고 재무제표 값 필터를 적용해 유사 문서만 추출했습니다. 

![AI응답 토큰화 처리](https://res.cloudinary.com/dcbaxgous/image/upload/q_auto/f_auto/image_dolvet.png)

토크나이저를 사용해 불용어 및 단순 수치 등 노이즈를 제거하여 컨텍스트를 압축했습니다. 또한, 각 항목 특성에 맞는 시스템 프롬프트를 동적으로 할당하고 내부 프롬프트를 영문으로 구성해 입력 토큰 소비를 최적화했습니다.

- System Prompt 예시
    
    ```json
    [SECTION ADDENDUM: BUSINESS RISK]
    
    LANGUAGE
    - Write ALL output in Korean.
    
    INPUTS (CONTEXT PAYLOAD — DO NOT ECHO)
    - CORP_NAME := <corpName>               // issuer name
    - IND_NAME  := <indutyName>             // industry / sector
    - WEB_RAG   := <webRagItems>            // JSON[] of {source, title, url, pubDate(YYYY-MM-DD), content}
    - DART_RAG  := <dartRagItems>           // JSON[] of {issuer, filingType, filedDate, section, snippet, url}
    - MAX_ITEMS := <maxItems>               // number of risk items to produce (default: 5)
    
    GOAL
    - Draft ONLY the “Business Risk” section: risks that arise from IND_NAME and the issuer’s business model/operations. Do not include content from other sections.
    
    SELECTION
    1) Use WEB_RAG as PRIMARY evidence. Score candidates by (impact × likelihood) and keep the top MAX_ITEMS.
    2) Use DART_RAG for structure/wording patterns and fact cross-checks (optional supporting citation).
    3) Exclude generic, economy-wide risks unless tightly linked to IND_NAME with concrete causal channels.
    
    WRITING (paragraphs only; no lists/tables/headings)
    - Each risk item MUST follow:
      a) One-sentence TOPIC LINE: “cause/condition → specific impact on CORP_NAME”.
      b) 1–2 follow-up paragraphs: evidence (figures/dates/policies/supply/demand/pricing), impact pathway, and current mitigation (if any).
      c) Each risk item MUST start with a Korean sequential marker such as "가.", "나.", "다.", ... at the beginning of the paragraph.
    
    - Evidence-based, precise, neutral. No hype, vagueness, or unfounded certainty.
    - When using an external fact, append ONCE at sentence end: `출처: SourceName(YYYY-MM-DD)`.
    - Include units and time context with numbers. Define technical terms once (in parentheses) on first use.
    
    OUTPUT FORMAT (strict)
    - Body text only in Korean.
    - Separate risk items with EXACTLY one blank line.
    - Do not introduce other sections (e.g., financial condition, dividends).
    
    DATA USE HINT
    - Prioritize IND_NAME signals and WEB_RAG keywords; enrich with DART_RAG snippets from similar issuers (same industry/financial profile).
    ```

## 성과 및 회고

### 성과

1. 증권신고서 초안의 약 70%를 자동 생성하는 AI 파이프라인을 구축했습니다.
2. 비동기 처리 및 아키텍처 개선을 통해 AI 응답 생성 시간을 기존 대비 65~85% 단축(30~40분 → 5~10분)했습니다.
3. 해당 프로젝트로 더존비즈온 최종 발표에서 **프로젝트 대상**을 수상했습니다.

### 회고

1. **컨텍스트 엔지니어링의 중요성**: Spring AI 도입 초기, 레퍼런스가 부족한 상황에서도 공식 문서를 분석하며 아키텍처를 설계했습니다. 이 과정에서 AI의 할루시네이션을 제어하고 정확도를 높이기 위해서는 단순 프롬프트 작성을 넘어선 '컨텍스트 엔지니어링' 기반의 시스템 설계가 핵심임을 배웠습니다.
2. **데이터 확장의 필요성**: 공개된 비상장 기업의 DART 보고서만 활용한 점은 아쉬움으로 남습니다. 기업 내부의 상세 재무 보고서, 차트 등 풍부한 비공개 데이터가 결합된다면 투자위험요소 등에서 더욱 고도화된 인사이트를 도출하는 에이전트로 발전할 수 있을 것입니다.
3. **인프라 고도화 제안**: 비용 문제로 Docker 기반 로컬 환경을 구축했으나, 향후 클라우드 기반 OpenSearch 서비스 및 RAGFlow와 같은 RAG 특화 오픈소스를 본격적으로 도입한다면 검색 성능과 유지보수 효율을 한 단계 더 끌어올릴 수 있을 것으로 판단됩니다.
