## 개요

RAG(검색 증강 생성) 기술을 응용한 AI 엔지니어링 역량 강화를 목적으로 기획된 프로젝트입니다. 1인 가구 및 고령층 증가로 반려동물을 키우는 인구가 늘어남에 따라 발생하는 관리의 어려움을 해결하고자 했습니다. 수의학 가이드라인 문서를 임베딩한 벡터 DB를 활용하여 반려견의 특성에 맞는 맞춤형 일정을 자동으로 생성하고, Google Calendar에 등록하여 체계적인 관리를 돕는 웹 애플리케이션 서비스입니다.

## 역할

- **AI 챗봇 및 RAG 파이프라인 구축**: LangChain 에이전트 구성, 대화 기록 유지를 위한 콜백 및 히스토리 구조 구현, 수의학 가이드라인 기반 벡터 DB(Pinecone) 연동.
- **사용자 맞춤형 일정 생성 로직 개발**: 반려견 등록 정보 및 RAG 검색 결과를 바탕으로 맞춤 일정 및 설명글 생성 파이프라인 구축.
- **프론트엔드 및 백엔드 연동**: Streamlit 기반 사용자 정보 및 강아지 정보 등록 UI/CRUD 구현.
- **외부 API 및 데이터 관리**: RDBMS 부재를 대체하기 위해 Google Drive API를 활용한 사용자 JSON 데이터 영속성 관리 및 Google Calendar 연동.

## Skills

- **Frontend**: Python Streamlit
- **Backend & AI**: Python, LangChain, RAG, Pinecone(Vector DB)
- **API & Auth**: Google OAuth, Google Drive API, Google Calendar API, DuckDuckGo Search API, SerpAPI
- **Data**: JSON 기반 파일 시스템 (Google Drive 연동)

## 주요 수행 내용 및 문제 해결

### 1. LangChain Agent 및 RAG 검색 툴 안정성 확보를 위한 Fallback 구조 설계

#### AS-IS
AI 에이전트에 DuckDuckGo 검색 툴을 연동하여 웹 검색 기능을 구현했으나, 검색 요청 제한(Rate Limit) 초과로 인한 오류가 빈번하게 발생하여 서비스 안정성이 저하되었습니다.

#### TO-BE
DuckDuckGo 검색 실패 시 예외 처리를 통해 다른 검색 엔진인 SerpAPI(Google Search)가 대체제로 동작하도록 Fallback(대체) 구조를 보완했습니다. 이를 통해 검색 제한 오류로 인한 챗봇의 응답 중단을 방지하고 안정적인 답변 산출 로직을 확보했습니다.

- AI LangChain Agent 코드
    
    ```python
    def init_agent_chain(memory):
        # 1) LLM
        llm = ChatOpenAI(
            model_name=OPENAI_API_MODEL,
            temperature=OPENAI_API_TEMPERATURE,
            streaming=True,
        )
    
        # 2) 검색 도구
        duck = DuckDuckGoSearchRun(name="DuckDuckGo")
        serp = SerpAPIWrapper(serpapi_api_key=SERPAPI_API_KEY)
        def search_fallback(q: str) -> str:
            try: return duck.run(q)    # 메인 검색툴
            except: return serp.run(q) # 예외처리 폴백
        search_tool = Tool.from_function(
            func=search_fallback,
            name="WebSearch",
            description="반려견 관련 정보를 검색할 때 사용합니다. (DuckDuckGo→Fallback Google)",
        )
    
        # 3) 전문 상담사 페르소나
        system_prompt = SystemMessagePromptTemplate.from_template(
            "당신은 10년 경력의 수의사이자 반려견 행동 전문가입니다. "
            "모든 반려견 관련 질문에 근거 기반으로, "
            "친절하고 이해하기 쉽게 한국어로 답변하세요."
        )
    
        # 4) 대화 프롬프트 조합
        prompt = ChatPromptTemplate.from_messages([
            system_prompt,
            MessagesPlaceholder(variable_name="chat_history"),
            HumanMessagePromptTemplate.from_template("{input}"),
        ])
    
        # 5) 에이전트 생성
        chat_agent = ConversationalChatAgent.from_llm_and_tools(
            llm=llm,
            tools=[search_tool],
            prompt=prompt,
        )
        return AgentExecutor.from_agent_and_tools(
            agent=chat_agent,
            tools=[search_tool],
            memory=memory,
            return_intermediate_steps=True,
            handle_parsing_errors=True,
        )
    ```

### 3. 입력 토큰 오버플로우 한계 극복 (동적 프롬프트 루프 처리)

#### AS-IS
여러 개의 맞춤 일정을 한 번의 프롬프트 요청으로 생성하려다 보니, LLM의 입력 토큰 한계(Overflow)를 초과하여 응답이 절단되거나 에러가 발생했습니다.

#### TO-BE
각 일정(토픽)마다 System 및 User 프롬프트를 동적으로 분리하여 개별 할당하고, 이를 루프(Loop)로 돌려 순차적으로 데이터를 얻어내는 방식으로 아키텍처를 변경했습니다. 개별적으로 산출된 결과 데이터들은 하나의 JSON 구조로 병합하여 사용자 데이터베이스(Google Drive)에 동기화한 뒤 캘린더에 성공적으로 연동했습니다.

## 성과 및 인사이트

### 사실 (Fact)

1. LangChain과 Pinecone을 활용해 수의학 문서를 임베딩하고, 반려견 특성 기반의 스케줄 자동 생성 시스템을 구축했습니다.
2. DB 구축의 대안으로 Google Drive와 연동하여 사용자의 강아지 정보 및 헬스노트 JSON 데이터를 안전하게 저장하고 동기화했습니다.
3. 사용자가 앱에서 일정을 생성하면 개인 스마트폰의 Google 캘린더 앱에 자동으로 동기화되는 환경을 구현했습니다.

### 의견 (Opinion)

1. **플랫폼 한계의 창의적 극복**: 모바일 애플리케이션 배포 경험이 없어 웹앱(Streamlit)으로 구현했으나, 대다수의 스마트폰에 설치된 Google Calendar와의 API 연동을 통해 모바일 알림 기능을 대체한 것은 현재 가진 기술적 제약을 창의적인 아이디어로 극복한 유효한 전략이었습니다.
2. **프롬프트 엔지니어링의 중요성**: 다양한 강아지 종류와 조건에 대한 처리를 하나의 프롬프트에 하드코딩한 점은 아쉬움으로 남습니다. 복잡한 조건을 처리할 때는 프롬프트를 세세하게 쪼개고 분기 처리하여 라우팅하는 방식이 LLM의 속도와 정확도를 높이는 데 훨씬 유리하다는 것을 깨달았습니다.

## 회고 및 개선 방향

### 회고
RDBMS를 배우기 전의 한계 속에서도 Google Drive API를 데이터베이스처럼 활용해 CRUD를 구현하고, 프론트엔드와 LLM 백엔드를 직접 연결해보며 AI 서비스의 전체 사이클을 훑어볼 수 있었던 의미 있는 프로젝트였습니다. 특히, AI가 분석한 데이터가 내 스마트폰의 캘린더에 실제 일정으로 꽂히는 과정을 직접 구현하며, AI가 사용자 일상에 실질적인 가치를 제공하는 방식을 체감할 수 있었습니다.

### 개선 방향 (Architecture & Data)

1. **프롬프트 라우팅 및 세분화**: 현재 1개의 프롬프트에 의존하는 하드코딩 로직을 개선하여, 강아지의 종이나 특정 질환 등 주요 조건에 따라 프롬프트 체인을 나누는 'Semantic Router' 또는 분기 체인을 도입한다면 응답 품질과 처리 속도를 개선할 수 있습니다.
2. **캐싱(Caching) 도입**: 동일한 반려견 정보와 가이드라인을 반복적으로 RAG에서 검색하는 과정에서 발생하는 지연 시간을 줄이기 위해, 빈번하게 질문하는 토픽에 대한 Context를 캐싱하고 재활용하는 구조가 필요합니다.
3. **이벤트 기반 캘린더 동기화**: 현재 버튼 클릭 등의 수동 트리거로 작동하는 캘린더 업데이트 방식을 개선하여, 일정 변경 및 추가 감지 시 Background에서 자동으로 반영되는 실시간 동기화 아키텍처로 고도화할 것을 제안합니다.
